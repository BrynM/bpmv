bpmv.js

Introduction

What's bpmv.js and why should I use it?

These are tasks that I find myself repeating over and over again. The rules for me including code in here are simple: nothing that tries to manipulate the DOM itself and things that mainly revolve around testing or altering different kinds of states or values.

I provide no promises that I will follow my own rules or that any of this code is fit for any purpose. This is a personal library and not a full-blown open source project. I will add and remove what I see fit. Consider yourself warned.

That said, I do hope that you can find something you like here. I would recommend that you use a descreet copy and do not load the file from a pulbic source (as I may refactor on a whim). Full Usage Documentation

Optional Browser Setup

By default, the global variable `bpmv` will be created (attached to `window`). If node.js is detected as the current environment, `exports.bpmv` will be created. Additionally, there are two (admittadly sloppy) global settings you can use to control bpmv.

If you do not wish to have bpmv called `bpmv`, you can set BPMV_VARNAME to any string you wish. For example: 

BPMV_VARNAME = 'validatorThingy';
// ...bpmv.js is loaded via >script< tag or some other method
validatorThingy.str( 'foo' ); // use it to test a string

At file/script load, the varialbe BPMV_ATTACH will be checked or "object-ness". If it is an object, you can use it to attach bpmv to a structure of your choice. For example: 

var BPMV_ATTACH = my.big.construct;
// ...bpmv.js is loaded via >script< tag or some other method
// loading the file _after_ setting BPMV_ATTACH is important!
// (I do realize that this pollutes the current scope some. Sorry.)
my.big.construct.bpmv.str( 'foo' ); // attached and ready for use

Used together, BPMV_ATTACH and BPMV_VARNAME should give you enough control to avoid any name clashing and perform any customization you'd like.

.arr ( dIsArr, okEmpty )

No, this doesn't test for piracy. bpmv.arr() tests if something is not just an object, but is an Array object and is not empty. It can also test if the thing optionally has some or a specific number of elements.

ParameterTypeExplanation
dIsArr mixed The value you'd like to test.
okEmpty mixed okEmpty If Boolean, is a flag for whether the array will be tested for emptiness. If an integer, will test the array for that exact length.
RETURN: boolean Will return true if an array with contents (or optionally passes okEmpty) and false if not or upon failure.
Examples
// these return false
bpmv.arr( [] );
bpmv.arr( [ 'booty' ], 0 );
bpmv.arr( [ 'booty', 'treasure' ], 1 );

// these return true
bpmv.arr( [], true );
bpmv.arr( [], 0 );
bpmv.arr( [ 'booty' ] );
bpmv.arr( [ 'booty' ], 1 );
bpmv.arr( [ 'booty', 'treasure' ], 2 );

.basename ( redOrBlue )

Get the basename of a filesystem path or URL.

ParameterTypeExplanation
redOrBlue string Some sort of filesystem path or URL. The path can be in either standard (slashes) or Windows (backslashes) format.
RETURN: string Returns the parsed dirname.
Examples
bpmv.basename( '/' ); // returns '/'
bpmv.basename( '/path/to/a/thing' ); // returns 'thing'
bpmv.basename( '/path/to/a/thing/' ); // returns 'thing'
bpmv.basename( 'C:\\' ); // returns 'C:\'
bpmv.basename( 'C:\\Program Files\\App\\thing' ); // returns 'thing'
bpmv.basename( 'C:\\Program Files\\App\\thing\\' ); // returns 'thing'
bpmv.basename( 'http://google.com' ); // returns 'http://google.com'
bpmv.basename( 'http://google.com/' ); // returns 'http://google.com'
bpmv.basename( 'http://google.com/mail' ); // returns 'mail'

.ccase ( hump, wspace, spit )

Convert a string to camelCase.

ParameterTypeExplanation
hump string The string to convert.
wspace mixed The whitespace character(s) you would like to remove for the conversion. If a character, it can be an entire string or several comma separated characters (no whitespace around your commas). wspace can also be an Array of characters. The default wspace is [ ' ', '_', '-', '\r', '\n' ].
spit boolean If spit is true, the first letter of the string will be capitalized.
RETURN: string Returns the camel cased version of your string. If the hump parameter is not a string, it will be returned as-is to prevent destruction ot loss.
Examples
bpmv.ccase( 'This is some stuff' ); // returns 'thisIsSomeStuff'
bpmv.ccase( 'This_is_some_stuff' ); // returns 'thisIsSomeStuff'
bpmv.ccase( 'This-is-some-stuff' ); // returns 'thisIsSomeStuff'
bpmv.ccase( 'This is-some_stuff' ); // returns 'thisIsSomeStuff'
bpmv.ccase( 'This is\nstuff', [ '\n' ] ); // returns 'this isStuff'
bpmv.ccase( 'This is_stuff.', ' ', true ); // returns 'ThisIs_stuff.'
bpmv.ccase( 'look its a class_name', null, true ); // returns 'LookItsAClassName'

.ego ( fans )

Generate a random valid html id using characters and a timestamp. This is the only bpmv.js function that takes an options object parameter.

ParameterTypeExplanation
fans object An options object. Defaults are listed below. 
var defaults = {
    // Length of the string to generate (see absLen for forced truncation).
  'len' : 12,
    // A prefix to apply to the ID string generated
  'prefix' : '',
    // The character set (string list of characters) to choose from for
    // generating the "random" portion of the ID.
  'charset' : '0123456789ABCDEFGHIJKLMNOPQRSTUVWXYZ',
    // If set to true, a delimiter and integer UNIX time stamp will be appended
    // to the generated ID string.
  'useTime' : true,
    // If set to true, the entire return string will be truncated to the len
    // setting. Note that this may truncate away desired portions such as
    // timestamps. (legacy and kind of useless)
  'absLen' : false,
    // The delimiter to use between portions of the ID string (prefix, id and
    // time stamp).
  'delim' : '_'
};
RETURN: string The generated ID.
Examples
// returns something like 'ABC12DEF34GH_1234567890123'
bpmv.ego();

// returns something like 'ABC_1234567890123'
bpmv.ego( { len : 3 } );

// returns something like 'mine_ABC12DEF34GH_1234567890123'
bpmv.ego( { prefix : 'mine' } );

// returns something like 'inimmieimmim_1234567890123'
bpmv.ego( { charset : 'mine' } );

// returns something like 'ABC12DEF34GH'
bpmv.ego( { useTime : false } );

// returns something like 'ABC'
bpmv.ego( { len : 3, absLen : true } );

// returns something like 'ABC12DEF34GH+1234567890123'
bpmv.ego( { delim : '+' } );

// returns something like 'mine+ABC12DEF34GH+1234567890123'
bpmv.ego( { prefix : 'mine', delim : '+' } );

// let's get fancy
var egoOptions = {
	  len     : 25
	, prefix  : 'BIG_NUM'
	, charset : '123456789'
	, useTime : false
}
bpmv.ego( egoOptions ); // returns something like 'mine+ABC12DEF34GH+1234567890123'

.find ( pin, stack, siv )

Find the property name for a given element in an object or array. The first encountered match is what will be returned, so be carefult that what you are looking for is unique! If the property is not found, null is returned.

ParameterTypeExplanation
pin mixed What you are looking for. Can be any valid value.
stack mixed The object or array you are looking in.
boolean siv Assume the pin is not === and instead is the keyname of what you're looking for. If this option is true, the value of the property will be returned if found.
RETURN: mixed The name of the property if found or null if not found.
Examples
function foo () {}
var thing = {
	  'func' : foo
	, 'bar'  : 'stuff'
	, 'baz'  : 2
	, 'boo'  : null
}
bpmv.find( foo, thing ); // returns 'func'
bpmv.find( 'stuff', thing ); // returns 'bar'
bpmv.find( null, thing ); // returns 'boo'

.incall ( soil, fertilizer, weeds, flo )

Increment all numeric values in either an array or the top level of an object by a given amount. Note that this will also increment numbers inside of strings.

ParameterTypeExplanation
soil mixed The array or object to increment values in.
fertilizer number The amount you'd like the numbers incremented. This can be a positive or negative value - including 0. Floats are perfectly valid, but integer values being incremented by floats will be rounded. To control the rounding behavior, see the flo parameter. Defaults to 1.
weeds boolean Increment numeric strings too. If false, only proper Number values will be incremented. Defaults to true.
flo boolean Normally, when an integer value is being incremented by a float, the integer will be rounded using Math.round(). To force them to be converted to floating point numbers, pass this parameter as true. Defaults to false.
RETURN: mixed Will return the reultant version of soil.
Examples
bpmv.incall( [ 1, 1.25, 0 ], 1.6 ); // returns [ 3, 2.85, 2 ]
bpmv.incall( [ 1, 1.25, 0, '2.2' ], 1.6 ); // returns [ 2.6, 2.85, 1.6, "2.2" ]
bpmv.incall( [ 1, 1.25, 0, '2.2' ], 1.6, true ); // returns [ 3, 2.85, 2 ]
bpmv.incall( [ 1, 1.25, 0 ], 1.6 ); // returns [ 3, 2.85, 2 ]

.toke ( vessel, stash, onMyCase, couch )

A simple tokenizer.

ParameterTypeExplanation
vessel string The string containing the tokens you wish to replace.
stash object An object with the tokens as key names and the replacements as the values as key/value pairs.
onMyCase boolean blahblahblah
couch object couch can some in three formats. First, as an object containing two keys, 'r' and 'l'. Secondly as an array of two strings. The first (0) will be assigned to the left side. If couch is just a string, it will be applied to both left and right delimiters. These are used to denote the beginning and ending of a token in the original string.
RETURN: string Returns the string with the tokens found replaced.
Examples
var st = '##This## is text to change. Case sensitivity should change ##this##.'
    , slugs = { 'this' : 'that' }
    , altDelims = { l : '|@', r : '@|' }
    , altSt = '|@This@| is text to change. Case sensitivity should change |@this@|.';

// returns 'that is text to change. Case sensitivity should change that.'
bpmv.toke( st, slugs );

// returns '##This## is text to change. Case sensitivity should change that.'
bpmv.toke( st, slugs, true );


// we are using alternate delimiters on our alternate string
// returns 'that is text to change. Case sensitivity should change that.'
bpmv.toke( altSt, slugs, null, altDelims );

// we are using alternate delimiters on our alternate string
// returns '|@This@| is text to change. Case sensitivity should change that.'
bpmv.toke( altSt, slugs, true, altDelims );

.wrapped ( gift, box, bow, bag )

Word wrap a string to a given length. If this length is passed by a single word, the word will NOT be broken and it will appear on its own "line".

ParameterTypeExplanation
gift string The string that you would like to wordwrap.
box string The "line" length you'd like the gift to be wrapped at. The default is 80 characters.
bow string The string you would like to use as a "line" terminator. The length of your "line" terminator is NOT accounted for when measuring "line" length, so the resultant true length of each "line" is the lengh plus the terminator length. This parm being a variable of any string value is the reason the word "line" is shown in quotes so much.
bag string This is leftmost whitespace to be prepended to each line. The length of the bag whitespace IS taken into account when measuring the length of a line. The default is an empty string.

FAQ

  • Are bpmv functions chainable?

    No. That's not really the desired functionality. Most of bpmv is for testing things and returning boolean values.

  • Are there unit tests?

    Yes there are! They are run live in a page that is here.

  • Can I alter bpmv.js?

    You read the local copy of the GPL v2 License, right? If your alterations are nice, send over a pull request.

  • Can I use bpmv in Adobe Creative Suite JS files?

    I've done it and so far haven't seen any issues. That said, it's not officially supported and I won't pass out sample code for it.

  • Can I use bpmv in node.js?

    You betcha. I do all the time. 

    var bpmv = require( __dirname+'/bpmv.min.js' ).bpmv;

  • SuperCoolTrendyThing.js does X way better! Why don't you do it like them?
